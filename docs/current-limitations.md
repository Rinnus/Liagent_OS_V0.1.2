# LiAgent OS Current Boundaries and Limitations

LiAgent OS is a working alpha runtime. The project already has real agent loops, task execution, governance controls, and multi-surface delivery, but the public promise is intentionally narrower than the long-term vision.

## What the project does promise today

- a local-first agent runtime that can also operate across hybrid local-cloud model services
- working CLI, Web, and Discord surfaces
- governed behavior with approvals, policies, budgets, and auditability
- a recommended first-run path through API bootstrap mode

## What it does not promise yet

- a zero-config consumer product
- frozen public APIs or stable integration contracts
- full local-model parity across every machine and hardware profile
- hidden or silent autonomy without visible governance controls

## Current rough edges

- local-model setups still require manual model assets and path validation
- long-running workflows are live but still being hardened for reliability and recovery
- optional integrations such as browser tooling, MCP, and Discord require extra dependencies and setup
- behavior, configuration defaults, and workflow details may still change during the alpha phase

## Best current operator stance

- start with API bootstrap mode first
- move into `local_private` or `hybrid_balanced` after the base runtime is working
- treat approvals, budget controls, and audit traces as core parts of the product, not optional extras
- expect the strongest local-first experience to keep improving over time rather than already being fully polished

## Good fit right now

LiAgent OS is a good fit if you want to explore:

- governed agent execution instead of one-shot chat
- local-first or hybrid model routing
- long-lived research, monitoring, and follow-through workflows
- personal assistant direction with explicit privacy and safety boundaries

## Not the right fit yet

LiAgent OS is not the right current choice if you need:

- a one-click hosted consumer assistant
- guaranteed stable APIs for external platform integrations
- zero-maintenance local-model setup on day one
- a system that acts broadly in the background without human review or governance
